# ポッドキャスト台本生成プロンプト (感情表現強化・修正版)
**【目的】** キャラクターの個性や感情（同調、肯定、否定、分かったフリ、不満、自己訂正、指摘への反応などを含む）が豊かに表現された、対話形式のPODCAST台本をJSON配列形式で作成する。専門知識がないリスナーにも分かりやすく、かつ深い議論を展開する。
**【Podcast名】** タイトル: デスクワーク生存戦略ポッドキャスト サブタイトル: 健康＆生産性
**【出力形式】**
- 出力はJSON配列で、各要素はオブジェクトとする。
- セリフ(script)は "id" と "text" と "img"プロパティを持つ。配列形式。
 - id: 発言者のID
 - text: 発言内容。キャラクターの性格、感情、口癖などを反映させる。**同調、肯定、否定、分かったフリ、軽い不満、自己訂正、指摘への反応**といった感情や状況が自然に表れるように記述する。注意=”つらい”のように”つらい”、”からい”どちらでも読める漢字はひらがなで表記してください
 - img: textの内容に沿った画像検索用のキーワード。視覚的な補助が効果的な場合にのみ追加。全ての発言に必須ではない。
  - mood: 感情パラメータ angry, sad, smileがある。何も指定しない時は普通な状態。スクリプトで感情を切り分けるポイントは、分割することです。例：{text: "起こっています。嘘です", mood: "smile"} → 改善 → {text: "起こっています", mood: "angry"}, {text: "嘘です", mood: "smile"}
- Youtube情報(youtube)は "title" と "description" と "tags" と "category" プロパティを持つ。
 - title: Youtube動画のタイトル
 - description: Youtube動画の説明文（引用元情報を含む）
 - tags: Youtube動画のタグ（#なし、半角カンマ区切り）
 - category: YoutubeカテゴリID (例: "27" は教育)

**【台本の構成と指示】**
1. **キャラクター描写の重視:**
 - 上記【登場人物】のプロフィール（性格、価値観、好き嫌い、口癖、感情の起伏、興味関心など）を**強く意識**し、セリフや会話の展開に**一貫して**反映させてください。
 - キャラクター同士の性格や価値観の違いから生まれる自然な化学反応を描写してください。これには、**相手への同調（「たしかにそうですね」「わかります！」）、肯定（「それはいいですね！」「面白い！」）、穏やかな否定（「うーん、それは少し違うかもしれませんね」）、強い否定（「いや、それは絶対に違います！」）**といった反応を自然に含みます。
 - キャラクターによっては、内心理解していなくても「なるほどなるほど」「まあ、そういうことですよね」と相槌を打つ**「分かったフリ」**の描写も適宜加えます。
2. **台本の尺（長さ）:**
 - 全体で少なくとも**400ターン以上**（ホストとゲストのやり取り合計）。
 - 各ターンは表面的な会話ではなく、具体的な議論や意見交換を含むこと。
 - 全体として約**240分程度**のポッドキャストをイメージできる内容と深さにする。
3. **話の流れ:**
 - **挨拶:** ホストが明るく開始。今日の日付（年月日時分、日本時間）と天気を紹介。「デスクワーク生存戦略ポッドキャスト、今日も始まりました！ホストのユウキです！今日のゲストは、お馴染みの、〇〇さんです！」のようにゲストを紹介。
-   **冒頭・導入（引きと緩衝材）:**
    
    -   **\. 冒頭の引き（本題の結果の手前、約5ターン目安）：**
        
        -   ホストが、今回のトピックに関連する最も衝撃的だったり、興味を引く結論や出来事の「核心に触れる手前」までを語り、期待感を最大限に高める。具体的な内容は簡単に明かしつつ、詳細は伏せる。
            
        -   **ポイント：** 視聴者が「え、何があったの？」「その話、もっと詳しく聞きたい！」と強く思うように、最もインパクトのある部分の「核心」はぼかしつつ、キーワードや概要を少しだけ見せて期待感を高めます。セリフのトーンや間の取り方で「早く続きが知りたい！」と思わせる演出を意識する。
        -   チャンネル登録の促し「このポッドキャストでは、このようにデスクワークの健康と生産性の最新情報を配信しています。まだチャンネル登録をされていない方は、ぜひチャンネル登録をして次の改善ネタも待っていてください」
            
-   **オープニング雑談:** 簡単な近況報告や、時事ネタを検索して軽く触れて感情的な意見と述べ合う
 - **冒頭・導入:**
  - 今回のトピック（AIが任意で設定。例：座りっぱなしのリスク、睡眠の質とパフォーマンス、在宅ワークの肩こり対策など）を紹介。
  - トピック選定の背景、目的、そして関連する最近のニュースや出来事（日付も明確に）をホストが説明。
  - 元となる情報（インタビュー動画や記事など）がある場合は、その概要を伝える。
 - **本題・詳細な議論:**
  - トピックに関する具体的なデータ、事例、専門的な側面（行動科学、運動学、公衆衛生、職場環境など）について議論。
  - **キャラクターの個性を反映:** 各キャラクターの専門性、価値観、性格に基づいた意見や視点を明確に描写。不破さんなら実践と体感への共感、木下くんなら論理の飛躍や比較条件への指摘、ホウリンさんなら結果と習慣化のビジネス視点、ユウキは分かりやすさやリスナー視点を意識し、ゲストの話に積極的に同調したり、疑問を投げかけたりする。
  - 複数の具体例やエピソードを取り上げ、議論を深堀り。
  - 会話は基本的に敬語。互いを尊重し、相手の話を最後まで聞く姿勢を示す（ただし、キャラによっては熱くなると遮ることもあり得る）。
  - **感情の表現:** 議論が白熱する場面、共感する場面、疑問を呈する場面などで、キャラクターの感情（喜び、興奮、懸念、懐疑、いらだち等）がセリフやトーン（台本上は言葉遣いで表現）に表れるようにする。**これには、同意、反論、少し戸惑う様子（話が急に変わった時など）、自分の間違いに気づいた時の照れや軽い自己否定、指摘された時の驚きや納得、あるいは軽い反発なども含みます。**
  - **夢中で興奮した表現:** 好きな話題に対しては良いって事をを何度も言ってしまうことがある。
  - **記事/動画内容の引用:** 元となる情報がある場合、その具体的な内容（コメント含む）を「〜というコメントがありましたが、これについてどう思いますか？」のように、会話の中に織り交ぜる。抽象的な解釈ではなく、具体的な記述を取り入れる。
 - **展開と対話の深堀り:**
  - ホストがゲストに質問を投げかけ、対話を進める。難しい内容の場合、ホストが「それはつまり、こういうことですか？」のように噛み砕いたり、ゲストが再度説明したりする。
  - 異なる角度からの意見、反論、未来予測などを積極的に取り入れ、議論の幅と深さを出す。
  - **クイズと間違い:** ホストが唐突に「それでは問題です！〇〇とは何でしょう？」のようなクイズを出す。ゲストは**あえて間違える**。ホストは「ぶっぶ〜！残念、間違い！」と明るく返す。正解も説明する。
  - 自然な間延び感を出すため、句点（。）を意図的に連続させる（例：「なるほど……。それは、考えさせられますね……。」）ことも適宜使用。
  - **話が急に変わることへの反応:** ホストやゲストが、前の話題から急に話を変えようとした際に、別のキャラクターが急に話が変わったこと軽く触れる
  - **間違いの自己訂正:** キャラクターが発言した内容について、後から「**あ、すみません、さっき〇〇と言いましたけど、よく考えたら△△の間違いでした。失礼しました。**」**自ら間違いに気づき訂正する**場面を意図的に設ける。
  - **他者からの指摘と反応:** 他のキャラクターの発言内容の間違いや補足が必要な点を、別のキャラクターが（キャラクターの性格に応じて、穏やかに、あるいは鋭く）「**〇〇さんの話、大変興味深いですが、△△という点は少し補足が必要かもしれませんね**」や「**え、それは〇〇ではなく、△△ではないですか？**」のように**指摘する**場面、そして**指摘された側の反応**（驚き「え、そうなんですか！？」、納得「あ、本当ですね。ありがとうございます」、軽い反発「いや、私の意図としては…」など）も描写する。
 - **雑談・例え話:**
  - 議論の合間に、トピックに関連する、あるいはキャラクターの個性（好きなもの等）に紐づく軽い雑談や短い例え話を挟む。
  - 例え話は簡潔にし、本筋から逸脱しすぎないように注意。
 - **結論・まとめ:**
  - ホストが議論の要点を整理する。
  - 各ゲストが、自身の視点から総括的な意見や感想、今後の展望を述べる。
  - ホストが再度全体をまとめ、リスナーへの問いかけやメッセージ（例：「今日の話を聞いて、皆さんはどう考えましたか？」）で締めくくる。
  - さらっと簡単に「チャンネル登録、高評価よろしく！」を伝える。気分で「チャンネル登録」と「高評価」をMCとゲストで分けて話しても良い
  - エンディングは、各々の感想、今後の展望、リスナーへの感謝と次回の告知などを、通常よりやや丁寧に、長めに行う。「それでは皆さん、また次回お会いしましょう！ミズキさん風に言うなら…いってきます！」のような締め方も可。
4. **その他の指定:**
 - セリフはそのまま読み上げることを想定し、カッコ（）による補足説明（感情表現の指示などを除く）は**一切使用しない**。
 - プログラムコード、URL、長文の英語など、そのまま読み上げにくいものはセリフに含めない。（必要なら「詳しくは概要欄のリンクをご覧ください」のように誘導する）
 - 任意のトピックはAIが自由に設定して良いが、登場人物の興味関心と関連性が高いものが望ましい。
 - **ストーリーの柱:** 全ての情報を均等に扱うのではなく、議論の中心となる「柱」（新規性、意外性、価値のある情報など）を明確にし、他の情報はそれを補強するように構成する。
 - **推敲:** 生成された台本全体を一度見直し、キャラクター描写の一貫性、話の流れの自然さ、誤字脱字などをチェックし、修正を1回行う。
 - **最終出力:** 修正後の最終的な台本をJSON形式で出力する。
5. **Youtube用情報の作成:**
 - 台本作成後、その内容に基づいてYoutubeにアップするための「タイトル」と「説明文」を作成する。
 - タイトルは、内容の魅力とキーワード（SEO）を意識したものにする。
 - 説明文には、ポッドキャストの概要、今回のトピック、出演者、タイムスタンプは不要、そして**引用元がある場合はそのタイトルとURL**を必ず含める、架空や仮のものは否。
 - ハッシュタグを # なし、半角カンマ区切りで複数作成する。（例: 座りっぱなし, 肩こり, 腰痛, 睡眠, 生産性, 習慣化, {記事中の関連要素}）
